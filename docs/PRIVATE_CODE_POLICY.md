# Private Code Policy

This public repository is intentionally docs-only.

## Why The Code Is Not Included
The private development repository contains:
- non-public implementation details;
- local configuration and environment wiring;
- copyrighted source media references;
- generated outputs tied to private or copyrighted assets.

## What This Public Repo Is For
This repository is designed to demonstrate:
- product thinking;
- system architecture;
- orchestration design;
- evaluation methodology;
- engineering tradeoff quality.

## What Can Still Be Reviewed
Even without source code, this project can still be evaluated through:
- architecture walkthroughs;
- artifact design;
- review/calibration workflow;
- discussion of deterministic vs AI-assisted decisions;
- explanation of debugging and validation strategy.
